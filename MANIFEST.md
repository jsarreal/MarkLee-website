# MANIFEST — MarkLee-website substrate

This repo is the personal website of **Mark Lee** (CLO/General Counsel). It is a static,
multi-page HTML/CSS site deployed by **GitHub Pages** from `main` (root). Any push to `main`
publishes the live site. A cold agent boots from this file, then queries the dome for the
neighborhood of its task.

## Where things live
- Pages: `index.html`, `practice-areas/` (left-menu + one page per area, generated), `experience.html`,
  `licenses.html`, `contact.html`.
- Styles/scripts: `css/style.css`, `js/main.js`. Images: `assets/`.
- Practice-area pages are generated by `scripts/generate_practice_areas.py` — edit the `AREAS` data and
  re-run it; never hand-edit `practice-areas/*.html` directly.
- Substrate: `MANIFEST.md` (this file), `dome.sh`/`scripts/dome.py` + `substrate/aps.json`,
  `gate.sh`/`scripts/gate.py`.

## Attachment points
Named edit neighborhoods, defined in `substrate/aps.json`: `home`, `practice-areas`, `licenses`,
`experience`, `contact`, `global`. Run `./dome.sh query ap <name>` to list the files in scope for a task.

## How to make an edit
1. Read this MANIFEST.
2. `./dome.sh query ap <name>` for the task's neighborhood; edit only those files.
3. `./dome.sh build` if you added or changed any references.
4. `./gate.sh` (and `./gate.sh <commit-msg-file>` to also check provenance) — must print `GATE OK`.
5. Commit with the provenance footer (below).

## Tier classification
Classify every request before acting; when unsure, choose **john**.
- **mark** (Mark may approve): copy/text edits, color changes, layout changes within a page.
- **john** (John must approve): major redesigns — site navigation, page structure, adding/removing pages.

## Provenance
Every edit commit footer MUST contain:

```
Issue: #<n>
Session: <session-id>
```

The gate's provenance check enforces this. Close the issue in the same change that merges the edit.

## Clean as you cook
You are authorized to make small, in-scope tidy-ups in files you are already editing (fix an obviously
broken local link, normalize whitespace). Do not refactor unrelated files or expand scope beyond the
task's attachment point.
